# Contributing to Moonity

First of all, thank you for your interest in contributing to Moonity! 🚀

## 📋 Requirements

Before submitting any contribution, make sure you follow these requirements:

### 1. Code Style

All code must follow the formatting rules defined in the `.editorconfig` file.

- Ensure your editor is configured to respect `.editorconfig`
- Run formatting tools before submitting your PR

### 2. Unity Version

All contributions must be developed and tested using:

> Unity 6.0 (6000.0.68f1)

Using a different version may introduce incompatibilities.

### 3. AI Usage Disclosure

If you used any AI tools (such as ChatGPT, Copilot, etc.), you must clearly state it in your Pull Request.
The AI disclosure requirement is for transparency only. Pull requests will be evaluated based on their quality and correctness, regardless of whether AI tools were used.

Include:
- Which tool was used
- What parts were generated

Example:

AI-assisted: Yes
Tool: ChatGPT
Scope: Helped with initial implementation of X feature

### 4. Commit Convention

Use the following prefixes:

- feat: new feature
- fix: bug fix
- docs: documentation
- refactor: code improvement
- chore: maintenance

Example:
feat: add hotreload system

---

## 🔄 Pull Request Guidelines

- Keep PRs focused and small (one feature or fix per PR)
- Write clear and descriptive commit messages
- Link related issues when applicable
- Add comments to explain non-obvious code
- Avoid unnecessary dependencies

## PR Description

- What was done:
- Why it was done:
- How to test:

---

## 🧪 Testing

- Test your changes before submitting
- Do not break existing functionality
- If applicable, include steps to reproduce/test

---

## 🐛 Issues

When opening an issue, please include:

- A clear description
- Steps to reproduce (if it's a bug)
- Expected vs actual behavior

---

## 📦 Code Quality

- Prefer readable and maintainable code over clever solutions
- Follow existing project structure and patterns

---

## ⚠️ Disclaimer

By contributing, you agree that your code may be modified or rejected if it does not meet project standards.
